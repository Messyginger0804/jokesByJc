## Goal

**Singular Focus:** Fix the low contrast issue between body background and popup container by changing the body background color to create clear visual distinction while meeting WCAG 2.1 AA requirements.

**Non-Goals:**
- Redesigning the entire color scheme
- Adding new UI components
- Changing text colors (already compliant)

## Scope & Assumptions

**In Scope:**
- Change body background color in `styles.css` line 12
- Ensure minimum 3:1 contrast ratio between body and container

**Out of Scope:**
- Text contrast changes (joke-container already uses #111827 on #f3f4f6 - compliant)
- Button color changes (already have sufficient contrast)
- Adding borders or shadows (unnecessary if background contrast is fixed)

**Assumptions:**
- The popup container color `rgba(120, 113, 113, 0.9)` should remain unchanged to preserve design intent
- A darker body background will create sufficient distinction

## Deliverables (DoD)

1. Updated `styles.css` with new body background color
2. Contrast ratio between body and container meets 3:1 minimum
3. Visual distinction is clear between elements

## Readiness (DoR)

- [x] Access to `styles.css` - confirmed
- [x] Current colors identified: body and container both use `rgba(120, 113, 113, 0.9)`
- [x] WCAG requirements understood: 3:1 for UI components

### Task Details

**T1: Change body background color**
- **File:** `styles.css` line 12
- **Change:** `rgba(120, 113, 113, 0.9)` → `rgba(55, 50, 50, 0.95)`
- **Rationale:** Darker background (~#373232) against container gray (#787171) provides ~2.1:1 contrast. For better compliance, using `rgba(40, 38, 38, 0.95)` (~#282626) achieves ~3.2:1 contrast ratio.
- **Acceptance Tests:**
  - [ ] Body background is visibly darker than container
  - [ ] Container boundaries are clearly distinguishable

**T2: Visual validation**
- **Acceptance Tests:**
  - [ ] UI elements are clearly distinguishable
  - [ ] Design remains visually appealing
