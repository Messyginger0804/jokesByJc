## Goal

**Reduce total avatar image size by at least 40%** (from ~650KB to ~390KB or less) without visible quality loss at the 120x120px display size.

**Non-goals:**
- Changing image format to WebP (requires manifest and JS updates)
- Creating responsive image sets
- Changing the visual design of avatars

## Scope & Assumptions

**In Scope:**
- Resize PNG images from 398x398 to 240x240 (2x for retina displays)
- Lossless PNG compression using pngquant/optipng
- Verify images load correctly in extension

**Out of Scope:**
- WebP conversion (adds complexity, requires JS changes)
- SVG conversion (avatars are raster artwork, not suitable)
- Icon optimization (different ticket)

**Assumptions:**
- 240x240 at 2x retina will maintain quality at 120x120 CSS display
- PNG compression tools available on system or can be installed
- No transparency issues with compression

## Risks & Mitigations

| Risk | Impact | Likelihood | Mitigation | Trigger |
|------|--------|------------|------------|---------|
| Quality loss visible | High | Low | Keep originals as backup, use 240px (2x) | User reports blur |
| Tool not available | Medium | Low | Multiple fallback tools (ImageMagick, pngquant, online) | Install fails |
| Size reduction < 40% | Medium | Low | Aggressive compression settings, smaller dimensions | After T3 |

## Alternative Option: WebP Conversion

If PNG compression doesn't achieve 40% reduction:
- Convert to WebP format (typically 25-35% smaller than PNG)
- Requires updating `popup.js` image paths from `.png` to `.webp`
- No manifest changes needed (wildcards already in place)
- **Defer to separate ticket if needed**
