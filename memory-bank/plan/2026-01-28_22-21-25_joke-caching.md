## Goal

**Implement single-joke caching using `chrome.storage.local` to provide instant display of last joke on popup open and offline fallback.**

Non-goals:
- Multi-joke history/queue (deferred)
- Pre-fetching in background.js (deferred)
- Rate limiting or deduplication (out of scope)

## Scope & Assumptions

### In Scope
- Cache the most recently fetched joke in `chrome.storage.local`
- Display cached joke immediately when popup opens (before API fetch)
- Use cached joke as fallback when API + jokes.json both fail
- Simple cache structure: `{ lastJoke: { setup, punchline, isOneLiner, timestamp } }`

### Out of Scope
- Multiple joke history
- Background pre-fetching
- Cache expiration/invalidation
- Analytics on cache hits

### Assumptions
- `chrome.storage.local` is available (Chrome extension context)
- Storage quota sufficient for single joke object (~1KB)
- User prefers seeing last joke immediately over waiting for fresh content

## Risks & Mitigations

| Risk | Impact | Likelihood | Mitigation | Trigger |
|------|--------|------------|------------|---------|
| Chrome storage API not available | High | Low | Wrap in try/catch, degrade gracefully | Storage call throws |
| Cache corruption | Medium | Low | Validate cache structure before use | Missing/malformed fields |
| Storage quota exceeded | Low | Very Low | Single joke < 1KB, well under 5MB limit | N/A |
